## Prerequisites

Before you begin, make sure you have the following available:

- **Python 3.10 or later** installed on your machine
- **MySQL 8+** running locally or on a remote host
- **SMTP server access** (for example, Gmail, Outlook, or your organization's mail relay)
- **Git** to clone the repository

---

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/BrandonCVale/AMH_SISTEMA_OFICIOS.git
    cd AMH_SISTEMA_OFICIOS
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    ```bash
    python -m venv venv
    source venv/bin/activate        # macOS / Linux
    venv\Scripts\activate           # Windows
    ```

    <Tip>Using a virtual environment keeps the project's dependencies isolated from your system Python installation.</Tip>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This installs Flask, Flask-Login, Flask-Mail, PyMySQL, python-dotenv, pypdf, and reportlab.
  </Step>

  <Step title="Create the .env file">
    Create a file named `.env` in the project root with the following content, replacing each value with your own:

    ```env
    # Application
    SECRET_KEY=your-secret-key-here

    # MySQL database
    MYSQL_HOST=127.0.0.1
    MYSQL_USER=your_db_user
    MYSQL_PASSWORD=your_db_password
    MYSQL_DB=amh_oficios
    DB_PORT=3306

    # Email (SMTP)
    MAIL_SERVER=smtp.example.com
    MAIL_PORT=587
    MAIL_USE_TLS=True
    MAIL_USERNAME=your_email@example.com
    MAIL_PASSWORD=your_email_password
    MAIL_DEFAULT_SENDER=your_email@example.com
    ```

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore` to prevent accidental exposure of credentials.
    </Warning>

    <Note>
      `MAIL_USE_TLS` must be the string `True` (capital T). The app evaluates it with `os.getenv("MAIL_USE_TLS") == "True"`.
    </Note>
  </Step>

  <Step title="Initialize the database">
    Create the MySQL database and run your schema script:

    ```bash
    mysql -u your_db_user -p -e "CREATE DATABASE IF NOT EXISTS amh_oficios CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
    mysql -u your_db_user -p amh_oficios < schema.sql
    ```

    <Note>
      The app connects using PyMySQL with `DictCursor`. The connection is opened per request and closed automatically at the end of each request context.
    </Note>
  </Step>

  <Step title="Run the development server">
    ```bash
    python run.py
    ```

    The app starts on `http://0.0.0.0:5000`. Open `http://localhost:5000` in your browser — you will be redirected automatically to the login page.

    <Warning>
      The server runs with `debug=True` by default. Do not use this configuration in production.
    </Warning>
  </Step>

  <Step title="Log in with your assigned credentials">
    Enter the username and password provided by your administrator. Each account is tied to one of four roles:

    | Role | What they do |
    |------|-------------|
    | **Gestor** | Creates and sends oficios |
    | **Subdirector** | Reviews and assigns oficios to JUDs |
    | **JUD** | Processes and responds to assigned oficios |
    | **Administrador** | Manages users and the full document archive |

    If no user accounts exist yet, ask your administrator to create one using the admin panel or the `crear_usuario.py` utility.
  </Step>

  <Step title="Process your first oficio">
    Once logged in, you will land on the dashboard for your role. If you are a Gestor:

    1. Click the link to create a new oficio (navigates to `/oficios/crear_oficio`).
    2. Select the destination area, fill in the folio, subject, and request description.
    3. Attach any supporting PDF or document files — PDF files are automatically stamped with an acuse de recibido.
    4. Submit — the system assigns a sequential folio number, saves the record, and notifies the Subdirector by email.

    The oficio is now live in the system and visible to the assigned Subdirector.

    <Tip>
      The system automatically stamps uploaded PDFs with an *acuse de recibido* that includes the folio number, date, and recipient. No manual stamping required.
    </Tip>

    For a detailed walkthrough based on your role, see the relevant guide:

        Manage users and monitor all oficios
      </Card>
    </CardGroup>
  </Step>
</Steps>
