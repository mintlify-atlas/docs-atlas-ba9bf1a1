## Usuario class

Defined in `app/models/usuario.py`. Inherits from Flask-Login's `UserMixin`, which provides the default implementations required by Flask-Login (`is_authenticated`, `is_active`, `is_anonymous`, `get_id`).

```python
class Usuario(UserMixin):
    def __init__(
        self,
        id_usuario: int,
        nombre_completo: str,
        correo_electronico: str,
        id_rol: int,
        contrasena_hash: str,
        id_area: int,
    ): ...
```

The constructor maps to the `usuarios` table. Flask-Login requires the identity attribute to be named `id`, so `id_usuario` is stored as `self.id`.

### Role properties

Each property is a computed boolean derived from `id_rol`. Use these throughout the application instead of comparing `id_rol` directly.

| Property | `id_rol` value | Description |
|---|---|---|
| `es_gestor` | `1` | Can create and send oficios |
| `es_subdirector` | `2` | Reviews incoming oficios and assigns them to JUDs |
| `es_jud` | `3` | Attends and responds to assigned oficios |
| `es_administrador` | `4` | Manages users and can delete oficios |

```python
@property
def es_gestor(self):
    return self.id_rol == 1

@property
def es_subdirector(self):
    return self.id_rol == 2

@property
def es_jud(self):
    return self.id_rol == 3

@property
def es_administrador(self):
    return self.id_rol == 4
```

## Oficio model functions

Defined in `app/models/oficio.py`. These functions are the only layer that executes SQL against the database.

<Note>
  Functions that accept a `cursor` parameter **do not manage their own connection or transaction**. They must be called from inside a `with conexion.cursor() as cursor:` block, where the calling service is responsible for `conexion.commit()` and `conexion.rollback()`. Functions without a `cursor` parameter open their own connection and manage it internally.
</Note>

### crear_oficio_db

```python
def crear_oficio_db(cursor, datos) -> int
```

Inserts a new oficio header row into the `oficios` table. Sets `id_estatus_actual = 1` (En Revisi√≥n) automatically.

**Parameters:**

- `cursor` ‚Äî Active transaction cursor from the service layer
- `datos` ‚Äî Dict with keys: `folio`, `folio_consecutivo`, `asunto`, `descripcion_solicitud`, `id_creador`, `id_asignado`, `id_area`

**Returns:** `cursor.lastrowid` ‚Äî the auto-generated `id_oficio` (int).

### guardar_documento_db

```python
def guardar_documento_db(cursor, id_oficio, id_usuario, nombre_real, ruta, tipo) -> None
```

Inserts a file record into `documentos_oficio`. Extracts the file extension from `nombre_real` automatically.

**Parameters:**

- `cursor` ‚Äî Active transaction cursor
- `id_oficio` ‚Äî ID of the parent oficio
- `id_usuario` ‚Äî ID of the user who uploaded the file
- `nombre_real` ‚Äî Original filename (e.g. `"solicitud.pdf"`)
- `ruta` ‚Äî Relative storage path (e.g. `"uploads/45/abc123_solicitud.pdf"`)
- `tipo` ‚Äî ENUM value: `"SOLICITUD"`, `"ANEXO_SOLICITUD"`, or `"RESPUESTA_OFICIAL"`

### asignar_oficio_a_jud_db

```python
def asignar_oficio_a_jud_db(id_oficio, id_jud, id_subdirector, instrucciones) -> bool
```

**Returns:** `True` on success, `False` if an exception occurred (with rollback).

### eliminar_oficio_db

```python
def eliminar_oficio_db(cursor, id_oficio) -> None
```

Hard-deletes an oficio and its dependent records in the correct order to satisfy foreign key constraints. Must be called within a service-managed transaction.

**Cascade order:**
1. `DELETE FROM historial_oficios WHERE id_oficio = ?`
2. `DELETE FROM documentos_oficio WHERE id_oficio = ?`
3. `DELETE FROM oficios WHERE id_oficio = ?`

<Note>
  Physical files on disk are **not** removed by this function. The service layer (`ServicioOficio.eliminar_oficio_total`) is responsible for deleting the `static/uploads/{id_oficio}/` directory after committing the DB transaction.
</Note>

## Usuario model functions

### crear_nuevo_usuario

```python
def crear_nuevo_usuario(datos) -> bool
```

Hashes the plain-text password in `datos["contrasena_hash"]` using `werkzeug.security.generate_password_hash` before insertion. Sets `activo = 1`.

**Parameters:** Dict with keys: `nombre_completo`, `correo_electronico`, `contrasena_hash` (plain text), `puesto`, `id_rol`, `id_area`.

**Returns:** `True` on success, `False` on any exception (with rollback).

---

### eliminar_usuario

```python
def eliminar_usuario(id_usuario) -> bool
```

Performs a soft delete by setting `activo = 0`. The user record is preserved to maintain referential integrity in `historial_oficios` and other tables.

**Returns:** `True` on success, `False` on any exception (with rollback).

---

### obtener_todos_los_usuarios

```python
def obtener_todos_los_usuarios() -> list[dict]
```

Returns all active users (`activo = 1`) with their resolved role name and area name via JOINs.

**Returns:** List of dicts with keys: `id_usuario`, `nombre_completo`, `correo_electronico`, `puesto`, `rol`, `area`.

---

### buscar_usuario_por_email

```python
def buscar_usuario_por_email(email: str) -> Optional[Usuario]
```

Queries `WHERE correo_electronico = ? AND activo = 1`. Returns a fully constructed `Usuario` object or `None` if not found or inactive.

### buscar_usuario_por_id

```python
def buscar_usuario_por_id(id_usuario: int) -> Optional[Usuario]
```

Same as `buscar_usuario_por_email` but queries by primary key. Returns `None` for inactive users.

### obtener_juds_por_area

Returns all active users with `id_rol = 3` belonging to the given area.

### obtener_subdirector_por_area

```python
def obtener_subdirector_por_area(id_area) -> dict | None
```

Returns the first active subdirector (`id_rol = 2`, `activo = 1`) in the given area, or `None` if none is found. Used by `ServicioOficio` to resolve the initial assignee when creating an oficio.
